Client Project Overview

Project Overview

  • Description: This web-app is designed to promote the Jamil Student Pantry. My client is heavily involved with the student pantry but wishes for more volunteers to be present to help out with day-to-day operations
  • Intended Users: All UNC Charlotte students as well as others in the Charlotte community wanting to help out a local food pantry
  • Overview of Content: The website will include a home page, services page, sign up for volunteering hours, About us, and Contact Us page

Page Design

Home Page

  • Purpose: Landing spot for the user. The user can find all the useful link from here and can navigate anywhere from the home page.
  • Audience: Customer, students
  • Content: An overview of all the other pages. Having links guiding the user to each page
  • Users will NOT be entering data here
  • This page will contain hyperlinks, buttons, and maybe dropdowns
  • When the user clicks a button on this page they will be taken to another page on the website. Since no data us being collected, no data will be processed

Services

  • Purpose: To inform the user of the services that the food pantry offers to people
  • Audience: Anyone wanting to learn more about the food pantry and potentially contribute to the operations of it
  • Content: It will show the services that the food pantry provides to the public and customers
  • Users wil NOT be entering data
  • The page will contains buttons, a slideshow and hyperlinks
  • When the user clicks on the slideshow buttons it will take them to the next image. Also hyperlinks will be present to take the user to the other pages. There will also be button to apply for volunteering hours. This button will take the user to a different page with more content.

Volunteering Hours

  • Purpose: To provide hours of volunteering to students, customers, and users
  • Audience: Anyone looking for volunteering hours for any reason
  • Cotent: A form for the user to fill out as well as a calendar showing the hours available to volunteer throughout the week
  • Users will be entering data on this page. They will be filling out a form to submit intent to volunteer at the food pantry
  • These fields may need validation to make sure the values entered are in the valid range
  • The page will contain some drop downs to show options for the user. As well as potentially some buttons to submit the form and provide other options
  • Actions on this page may include: processing the user data when the form is submitted and validating that the form is correctly filled out

About Us

  • Purpose: To provide an overview of what the food pantry is for, what we do, how we do it, when we operate, and why we exist
  • Audience: Any users trying to learn more about our organization and what we stand for
  • Content: Who we are, what we do, why we do it, where we operate, how we operate, and why we run the food pantry
  • Page will NOT be asking for user input
  • This page will contain hyperlinks and potentially some buttons to read more about our organization
  • clicking the buttons will show more information for the user. The other links will navigate to another page

Helping the Communnity

  • Prupose: To advocate for how we help the Charlotte community and those that need help. It also shoows pride in the community and a sense of belonging to those that need it.
  • Audience: This page will be for the user to learn more about us
  • Content: A simple slideshow and other descriptions clearly showing how we help the Charlotte community
  • No Data input on this page will be required
  • The page will contain buttons for the slideshow
  • Actions on this page include: clicking the slideshow button to go to the next page and hyperlinks navigating to the other pages

Contact Us

  • Purpose: To provide the user with a method of contacting us and asking questions
  • Audience: The customer/user to contact us
  • Content: A form for the user to fill out containing information for them to contact us with any questions they may have
  • Users will be entering data on this page
  • These data fields may need to be validated to make sure they are in the correct range
  • This page will contain some buttons to submit the form
  • Actions on this page include data being processed and navigating to other pages

Dynamic Functionality

Slideshow

A slideshow will be implemented on two pages to showcase some information about the organization.The pages that they will be on are the "Helping the Community" and "Services" page. I learned about the slideshow in the last assignment we had at this link from W3 Schools.

Forms

Forms will also be implemented on the page for the user to input there information. They will be implemented on the "Contact Us" and "Volunteering Hours" page. I learned about the form from FCC activities and the activity we had earlier in the year on Canvas

Header and Footer Component

The header and footer components will be included using javascript. That way they don't need to be added to each page individually and can be changed when I change the component. This will be placed on all the pages.
